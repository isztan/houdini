Houdini

A simple collapse-and-expand widget.

Download Houdini

GitHub Project       Getting Started

How It Works

You can use either an <a> or a <button> element to toggle to the collapsed content.

A little snippet of jQuery matches the data-target value to the ID of the collapsed element. Then, it adds or removes an .active class from the collapsed element.

CSS shows or hides the content based on the presence of that class.

Example 1

Toggled with a link and collapsed by default.

Toggle 1

Now you see me, now you don't.

<a class="collapse-toggle" data-target="#collapse1" href="#">Toggle 1</a>

<div class="collapse" id="collapse1">
    Now you see me, now you don't.
</div>

Example 2

Toggled with a button and expanded by default.

For <button> elements, use the CSS3 data-target attribute. The use of href with <button> element is not valid HTML.

To show content by default, add the .active class to the collapsed content.

Viola!

<button class="collapse-toggle" data-target="#collapse2">A button this time</button>

<div class="collapse active" id="collapse2">
    Viola!
</div>

Progressively Enhanced

By default, collapse-and-expand toggles are hidden and all collapsed content is visible.

Houdini includes the JS Accessibility snippet, which adds a .js class to the <body> on page load. This activates Houdini, showing the toggles and hiding collapsed content.

Requires jQuery

Houdini requires jQuery.
